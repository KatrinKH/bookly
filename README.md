# 📚 Bookly

Мобильное приложение для управления личной библиотекой с читалкой, заметками и статистикой чтения.

Тема итоговой аттестационной работы: **«Приложение для управления личной библиотекой с читалкой, заметками и статистикой чтения»**.

---

## Описание проекта

Bookly — это мобильное приложение, которое позволяет пользователю:

- загружать книги в формате **PDF** и **EPUB**;
- читать их во встроенной читалке с сохранением прогресса;
- **автоматически** фиксировать дату начала и завершения чтения каждой книги;
- получать **обложку книги автоматически** (извлекается из самого EPUB-файла) или выбрать её вручную из альбома фото, а также сменить в любой момент;
- делать **заметки** прямо во время чтения, привязанные к конкретной странице;
- после завершения книги ставить **оценку** (1–5) и отмечать, понравилась она или нет;
- смотреть **статистику** прочитанного — по месяцам, сезонам и годам: количество книг, время чтения в часах, любимые жанры;
- переключать **светлую и тёмную тему** оформления.

Приложение реализовано как полноценное full-stack решение:

- **Frontend** — мобильное приложение на **Flutter** (Android/iOS).
- **Backend** — REST API на **Node.js + Express**.
- **База данных** — **PostgreSQL**.
- **Авторизация** — на основе **JWT-токенов**.

## Функциональность

### Авторизация
- Регистрация нового пользователя (email, пароль, имя).
- Вход в систему.
- Хранение токена на устройстве — повторный вход не требуется при перезапуске приложения.
- Выход из аккаунта.

### Библиотека
- Загрузка книги (файл PDF/EPUB + название, автор, жанр).
- Список книг с фильтром по статусу: все / читаю / прочитано / не начато.
- Карточка книги с обложкой, статусом, оценкой и прогресс-баром чтения.

### Обложки книг
- Для EPUB — обложка **автоматически извлекается из самого файла** при загрузке (поддерживаются оба стандарта: EPUB2 через `meta name="cover"` и EPUB3 через `properties="cover-image"`).
- Возможность выбрать свою обложку из альбома фото при добавлении книги.
- Возможность **сменить обложку** в любой момент на экране деталей книги.
- Если обложки нет — показывается иконка-заглушка по формату файла.

### Читалка
- Встроенный просмотрщик PDF (`syncfusion_flutter_pdfviewer`) с сохранением текущей страницы на сервере.
- Встроенный просмотрщик EPUB (`flutter_epub_viewer`, вертикальная прокрутка) с локальным кэшированием файла — повторное открытие книги происходит мгновенно, без повторного скачивания.
- При первом открытии книги автоматически фиксируется **дата начала** чтения.
- Автоматическое открытие и закрытие **сессии чтения** при входе/выходе из читалки — используется для подсчёта реального времени чтения в статистике.

### Заметки
- Добавление заметки к книге с привязкой к номеру страницы.
- Просмотр всех заметок книги.
- Удаление заметки.

### Завершение книги
- Отметка книги прочитанной — фиксируется **дата завершения**.
- Оценка от 1 до 5 звёзд.
- Отметка "понравилось / не понравилось".

### Статистика
- Сводные показатели: всего прочитано книг, суммарное время чтения (часы/минуты).
- График количества прочитанных книг с переключением периода: **месяц / сезон / год**.
- Список любимых жанров (по количеству прочитанных книг в каждом).

### Профиль и настройки
- Просмотр данных профиля (имя, email).
- Отдельный экран **Настройки**.
- Переключение темы оформления: **светлая / тёмная**, выбор сохраняется на устройстве.

### Оформление
- Авторская тёплая цветовая палитра (savory sage, avocado smoothie, blush beet, peach protein, oat latte, honey oatmilk, coconut cream) для светлой темы и приглушённые тёмно-оливковые тона для тёмной.
- Шрифт **Lobster** для основного текста, **Pacifico** для названия бренда «Bookly».
- Заголовки страниц центрированы.

## Как запустить проект

### 1. Backend

**Требования:** Node.js 18+, PostgreSQL 14+.

```bash
cd backend
npm install
```

Создайте файл `.env` на основе примера и укажите свои данные для подключения к PostgreSQL:

```bash
cp .env.example .env
```

Откройте `.env` и при необходимости поменяйте `DB_USER`, `DB_PASSWORD`, `DB_NAME` — они должны совпадать с вашей локальной настройкой PostgreSQL. Также рекомендуется сразу заменить `JWT_SECRET` на собственную случайную строку.

Создайте базу данных (если её ещё нет):

```bash
createdb bookly
```

Примените схему базы данных (создаст все необходимые таблицы):

```bash
npm run migrate
```

Запустите сервер:

```bash
npm run dev
```

При успешном запуске в терминале появится сообщение `Bookly API запущен на порту 3000`. Проверить работу можно открыв в браузере `http://localhost:3000/health` — должен вернуться JSON `{"status":"ok", ...}`.

### 2. Frontend (Flutter)

**Требования:** Flutter SDK 3.3+, Android Studio или VS Code с плагином Flutter, эмулятор Android/iOS или физическое устройство.

```bash
cd frontend
flutter pub get
```

**Важно:** перед запуском укажите правильный адрес backend в файле `lib/services/api_config.dart`:

- Android-эмулятор → `http://10.0.2.2:3000/api` (уже стоит по умолчанию);
- iOS-симулятор → `http://localhost:3000/api`;
- физическое устройство → `http://<IP-адрес-компьютера-в-локальной-сети>:3000/api` (компьютер с backend и телефон должны быть в одной Wi-Fi сети).

**Для iOS** также потребуется разрешение на доступ к фото в `ios/Runner/Info.plist`:

```xml
<key>NSPhotoLibraryUsageDescription</key>
<string>Bookly нужен доступ к фото, чтобы вы могли выбрать обложку для книги</string>
```

Запустите приложение:

```bash
flutter run
```

При первом запуске Flutter предложит выбрать устройство (эмулятор или подключённый телефон).
